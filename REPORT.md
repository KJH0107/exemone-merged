# exemONE 인터랙티브 기능 가이드 앱 — 기획서 및 보고자료

## 1. 프로젝트 개요

### 배경

exemONE DB 모니터링 플랫폼은 기능이 다양하고 화면 구조가 복잡해, 신입 교육이나 외부 고객 발표 시 구두 설명만으로는 기능 전달이 어렵다는 문제가 있었습니다.

기존에는 PDF 또는 화면 캡처 자료로 기능을 소개하였으나, 실제 화면 흐름을 체감하기 어려워 이해도 및 기억 전달력이 낮았습니다.

### 목적

실제 exemONE UI와 동일하게 구현된 인터랙티브 환경에서, 기능 설명과 퀴즈·행동 확인을 결합한 **자기주도형 학습 가이드 앱**을 제공합니다.

### 활용 대상

| 대상 | 활용 방식 |
|---|---|
| 신입 구성원 | 입사 초기 온보딩 — 실습형 기능 학습 |
| 외부 고객 / 발표 | 데모 환경 없이 인터랙티브 기능 시연 |
| 기존 구성원 | 신규 기능 업데이트 시 자체 학습 |

## 2. 핵심 기능

### 2-1. 우측 가이드 패널 (360px)

- 현재 화면의 각 UI 요소를 클릭하면 해당 기능 설명이 우측 패널에 표시
- 기능 설명 → 사용 팁 → 챌린지(퀴즈/행동 확인) 순으로 구성
- 패널을 열면 메인 콘텐츠 영역이 자동으로 밀려나는 구조 (오버레이 없음)

### 2-2. 챌린지 시스템

각 기능마다 3단계로 구성:

```
기능 설명 읽기
    ↓
객관식 퀴즈 응답 (정답 즉시 피드백)
    ↓
직접 해보기 (화면에서 행동 확인 → 완료 감지)
```

### 2-3. 커버리지 (페이지별 기능 수)

| 페이지 | 기능 수 | 가이드 연결 |
|---|---|---|
| 홈 | 6개 | 기획 완료, 연결 예정 |
| 데이터베이스 오버뷰 | 4개 | ✅ 완료 |
| 인스턴스 상세 (8탭) | 8개 | ✅ 완료 |
| 성능 분석 | 기획 중 | 예정 |
| 알람 | 기획 중 | 예정 |

### 2-4. 장애 분석 플로우 가이드

오버뷰 → 인스턴스 클릭 → 액티브 세션 탭까지 이어지는 **장애 대응 시나리오**를 단계별로 안내합니다. 단순 기능 소개를 넘어 실제 업무 흐름을 익힐 수 있습니다.

## 4. 단일 HTML 변환 계획

### 전환 목적

현재 Next.js 앱은 서버 실행이 필요해 외부 공유·오프라인 시연에 제약이 있습니다.
단일 HTML 파일로 변환하면:

- 서버 없이 파일 더블클릭만으로 실행
- 이메일·USB·공유 드라이브로 즉시 배포 가능
- 외부 발표 환경에서 인터넷 없이 시연 가능

### 변환 범위

```
단일 index.html
├── 모든 CSS 인라인
├── 모든 JS 번들 인라인
├── 이미지 Base64 인라인
└── ECharts 번들 포함
```

### 변환 제약 및 대응

| 제약 | 대응 방안 |
|---|---|
| Next.js SSR/라우터 제거 | 탭 전환 기반 SPA로 재설계 |
| ECharts 번들 크기 | 사용 차트 타입만 트리쉐이킹 |
| 폰트 로컬 파일 | Base64 인라인 또는 시스템 폰트 폴백 |

## 5. 진행 계획

### 단계별 로드맵

| 단계 | 내용 | 상태 |
|---|---|---|
| Phase 1 | 데이터베이스 오버뷰 + 인스턴스 상세 가이드 구현 | ✅ 완료 |
| Phase 2 | 홈 / 성능 / 알람 페이지 가이드 연결 | 진행 중 |
| Phase 3 | 단일 HTML 변환 + 오프라인 배포 | 예정 |
| Phase 4 | 외부 발표 / 신입 교육 실전 적용 | 예정 |

### Phase 2 세부 항목

- 홈 페이지: 상태 요약 카드 / 화면 커스텀 / 최근 방문 대시보드 / 실시간 알람 등 6개 기능 GuideHighlight 연결
- 성능 분석: Scatter / Trend 기능 설명 + 챌린지 구성
- 알람: 알람 규칙 / 알람 현황 기능 설명 구성
